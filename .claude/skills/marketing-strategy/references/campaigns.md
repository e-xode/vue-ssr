# Campaigns — Vue SSR Starter Kit

The frame for planning a campaign and the running calendar. A campaign is a decision; the `content`
agent produces the FR+EN assets that execute it.

## Campaign frame (fill one per campaign)

- **Name**:
- **Objective**: GitHub stars / npm installs / forks / credibility / services inbound.
- **Primary audience**: which segment from `positioning.md`.
- **Channel(s)**: where it runs (see `gtm.md` for the channel mix).
- **Offer / hook**: the single message (no price — the kit is free; lead with time saved or the
  AI-ready config).
- **Assets needed**: posts, article, README copy, visuals (hand to content + design).
- **Timeline**: start, key beats, end.
- **Success metric**: the one adoption number that says it worked (e.g. +N stars, +N npm installs).

## Standing campaign themes

- **AI-ready out of the box** — the curated Claude Code fleet/skills/rules/hooks ship with the kit. The
  flagship differentiator story; targets technical leads and Vue developers.
- **Skip the plumbing** — auth, i18n, admin, SSR, security, CI, Docker already wired. Targets solo
  developers and agencies under time pressure.
- **Reference implementation** — read real Vue 3 SSR + Express + locale-routing code. Evergreen
  educational angle; targets Vue developers and drives organic discovery.
- **Free and MIT** — production-grade, no paywall, no lock-in. Trust/credibility angle across all segments.
