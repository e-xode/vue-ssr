# Go-to-market — Vue SSR Starter Kit

The channel mix and launch sequencing. Strategy lives here; execution is delegated.

## Channel mix

| Channel | Role in funnel | Owner of execution |
| --- | --- | --- |
| GitHub repo + README | Primary storefront; convert discovery to a star/clone | content (README copy) + design (visuals) |
| npm package page + description/keywords | Discovery via package search; convert to install | content (description) + seo (keywords) |
| LinkedIn posts/articles (`src/assets/linkedin/`) | Reach, top-of-funnel awareness, the tooling story | content |
| dev.to / technical articles | Educational reach, SEO, credibility | content |
| Reddit r/vuejs and Vue communities | Peer discovery, authentic word-of-mouth | content |
| Landing / contact page copy (i18n) | Convert visitors to a clone/star or a services enquiry | content (copy via translate) + design |
| SEO (sitemap, meta, hreflang, npm keywords) | Organic discovery | seo skill |

The README and the npm description are the two highest-leverage surfaces: they are the storefront a
developer judges in seconds. Prioritize them.

## Funnel (adoption to services)

1. **Discover** — SEO, LinkedIn, dev.to, Reddit r/vuejs, npm/GitHub search.
2. **Try** — clone/install; production defaults and the AI-ready config build trust.
3. **Adopt** — use it on a real project; star/fork (the adoption signal we optimize for).
4. **Advocate** — recommend it; word-of-mouth in Vue communities.
5. **Enquire** — inbound to E-XODE services (the off-platform conversion event).

## Launch sequencing (for a new feature or page)

1. Decide positioning and audience (this skill).
2. Brief content + design; produce FR+EN copy and visuals.
3. Ensure the page is in the sitemap and has meta/hreflang (seo skill); refresh npm keywords if relevant.
4. Announce via LinkedIn + dev.to + the relevant Vue community.
5. Measure against the campaign's adoption success metric.
