## ðŸ”„ How Documentation is Kept Updated

1. **On Code Changes**: Update related docs
2. **On New Features**: Add to CHANGELOG.md
3. **On Breaking Changes**: Update migration guide
4. **On User Issues**: Add to TROUBLESHOOTING.md
5. **On New Patterns**: Add to DEVELOPER_GUIDE.md

### Documentation Standards

- Clear, concise language
- Real examples included
- Action-oriented (how to, not what is)
- Bilingual when possible
- Keep README.md as landing page
- One doc per major topic
- Cross-link between related docs
- Use headings for scannability
